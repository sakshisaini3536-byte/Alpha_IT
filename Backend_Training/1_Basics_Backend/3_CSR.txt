1️⃣ What is Client Side Rendering?
📘 Simple Definition

Client Side Rendering (CSR) means:
The browser downloads a small HTML file, then JavaScript (React) builds and updates the entire UI inside the browser.

👉 Rendering happens on the client (browser), not on the server.

2️⃣ Real-Life Analogy (Very Important)
🏗️ IKEA Furniture Example

Server → Sends all parts + instruction manual

Browser → Assembles everything step by step

You don’t receive a fully built table;
you build it yourself.

✔ Server gives resources
✔ Client does the work

3️⃣ How CSR Works (Step by Step)
1. User opens website
2. Server sends empty HTML
3. Browser downloads JS bundle
4. React runs in browser
5. React calls APIs
6. React renders UI

4️⃣ What Server Sends in CSR
index.html

<!DOCTYPE html>
<html>
  <head>
    <title>React App</title>
  </head>
  <body>
    <div id="root"></div>
    <script src="/main.js"></script>
  </body>
</html>

7️⃣ Key Characteristics of CSR

| Feature       | CSR              |
| ------------- | ---------------- |
| Rendering     | Browser          |
| Initial HTML  | Almost empty     |
| JS dependency | Very high        |
| SEO           | Weak             |
| Page load     | Slower initially |
| Interactivity | Very fast        |

8️⃣ Where CSR is Used in Real Life

✔ Dashboards

✔ Internal tools
✔ SaaS products

Gmail

Admin portals

9️⃣ CSR — Advantages

✅ Rich interactivity
✅ Fast UI updates
✅ Less server load
✅ Perfect for logged-in apps

🔴 CSR — Disadvantages

❌ Slow first load
❌ SEO problems
❌ Blank screen until JS loads

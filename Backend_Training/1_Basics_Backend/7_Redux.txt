🔶 1️⃣ What is Redux? (WHY it exists)
❌ Problem (Before Redux)

In large apps:

Many components need the same data

Data updates must be predictable

Context becomes messy for large state + async logic

Example problems:

Auth state

Cart data

User profile

Notifications

🧠 Real-life analogy

🏢 Company Notice Board

Instead of every employee telling everyone else updates

One central board shows the latest information

👉 Redux = central data store

🔶 2️⃣ What is Redux Toolkit? 

Redux Toolkit is the official, recommended way to use Redux.

It solves:

Boilerplate

Complexity

Confusion


🔹 3️⃣ Redux Store
📌 Reference

The store is the central place where all global state lives.

🧠 Real-life example

🏦 Bank vault:

All money is stored in one place

Employees don’t keep money personally

📌 Usage

Created once

Provided to entire React app

📌 Returns

A global object holding application state

Code :- store.js

import { configureStore } from "@reduxjs/toolkit";
import counterReducer from "./counterSlice";

console.log("🏪 Creating Redux Store");

const store = configureStore({
  reducer: {
    counter: counterReducer,
  },
});

export default store;

🔹 4️⃣ Slice (MOST IMPORTANT CONCEPT)
📌 Reference

A slice is a piece of the global state with logic to update it.

🧠 Real-life example

📁 File folder:

One folder for Accounts

One folder for HR

One folder for Inventory

Redux Slice = one folder

📌 Usage

Defines initial state

Defines reducers (how state changes)

Auto-generates actions

📌 Parameters

name → slice name

initialState → starting data

reducers → update logic

📌 Returns

Reducer function

Action creators

🧪 Code: counterSlice.js

import { createSlice } from "@reduxjs/toolkit";

console.log("📦 Counter slice created");

const counterSlice = createSlice({
  name: "counter",
  initialState: {
    value: 0,
  },
  reducers: {
    increment: (state) => {
      console.log("➕ Increment reducer called");
      state.value += 1;
    },
    decrement: (state) => {
      console.log("➖ Decrement reducer called");
      state.value -= 1;
    },
    reset: (state) => {
      console.log("🔄 Reset reducer called");
      state.value = 0;
    },
  },
});

export const { increment, decrement, reset } = counterSlice.actions;
export default counterSlice.reducer;


🔹 5️⃣ Provider (Connecting Redux to React)
📌 Reference

Provider makes Redux store available to all components.

🧠 Real-life example

📡 Wi-Fi router:

Once connected, all devices can access internet

🧪 Code: main.jsx

import React from "react";
import ReactDOM from "react-dom/client";
import App from "./App";
import { Provider } from "react-redux";
import store from "./store";

console.log("🔌 Redux Provider wrapping App");

ReactDOM.createRoot(document.getElementById("root")).render(
  <Provider store={store}>
    <App />
  </Provider>
);

🔹 6️⃣ useSelector (READ data)
📌 Reference

useSelector reads data from Redux store.

🧠 Real-life example

📖 Reading notice board:

You don’t change it

You just read information

📌 Parameters

Function that receives entire store state

📌 Returns

Selected piece of state

🔹 7️⃣ useDispatch (UPDATE data)
📌 Reference

useDispatch sends actions to Redux store.

🧠 Real-life example

📨 Sending request to manager:

“Please increment count”

📌 Returns

Dispatch function

🔹 8️⃣ Full React Component Using Redux
🧪 Code: Counter.jsx

import { useSelector, useDispatch } from "react-redux";
import { increment, decrement, reset } from "./counterSlice";

function Counter() {
  const count = useSelector((state) => state.counter.value);
  const dispatch = useDispatch();

  console.log("🎯 Counter rendered | count =", count);

  return (
    <div>
      <h2>Redux Counter</h2>
      <p>Count: {count}</p>

      <button onClick={() => dispatch(increment())}>
        Increment
      </button>

      <button onClick={() => dispatch(decrement())}>
        Decrement
      </button>

      <button onClick={() => dispatch(reset())}>
        Reset
      </button>
    </div>
  );
}

export default Counter;
